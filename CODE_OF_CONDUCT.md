# Code Of Conduct

This repository aims to be useful, calm, and precise.

## Expected Behavior

Participants are expected to:

- communicate respectfully and directly
- assume good faith until there is strong evidence otherwise
- focus discussion on the skill, the kernel, and the actual behavior of the repository
- explain disagreements in concrete terms
- help keep issues and pull requests actionable

## Unacceptable Behavior

The following is not acceptable:

- harassment or intimidation
- personal attacks
- hateful or discriminatory language
- deliberate disruption of issues, discussions, or pull requests
- bad-faith misrepresentation of another person's work or intent
- spam or low-effort noise that makes collaboration harder

## Scope

This code of conduct applies to:

- issues
- pull requests
- discussions
- code review
- other public interactions around this repository

## Enforcement

If a participant repeatedly makes collaboration unsafe or unproductive, maintainers may edit, lock, hide, or remove content, or restrict participation.

## Reporting

If there is a conduct problem, report it through GitHub with only the detail needed to make the issue understandable, or contact the repository owner through GitHub profile channels.
